怎么在Hexo的博文中插入图片

发表于 2017-11-06 | 分类于 开发笔记 | 本文总阅读量 次

Hexo

安装

1213214123

  • Node.js

如果您的电脑中已经安装上述必备程序，那么恭喜您！接下来只需要使用 npm 即可完成 Hexo 的安装。

$ npm install -g hexo-cli

完成上述操作Hexo就算安装完成了，非常简单。

建站

参见官方文档地址：https://hexo.io/zh-cn/docs/setup.html

安装 Hexo 完成后，请执行下列命令，Hexo 将会在指定文件夹中新建所需要的文件。

$ hexo init <folder>
$ cd <folder>
 $ npm install

新建完成后，指定文件夹的目录如下：

.
├── _config.yml
├── package.json
├── scaffolds
├── source
| ├── _drafts
| └── _posts
└── themes

_config.yml 网站的配置信息，您可以在此配置大部分的参数。
关于配置官网也给出了非常详细的说明：https://hexo.io/zh-cn/docs/configuration.html

需要特别注意的点是： 属性设置后面的：需要有空格再填写内容！！！

写作

默认情况下生成的时候是没有categories和tags的总目录的，这时候需要我们人为的添加。
在source目录里面新建两个文件夹分别命名为categories和tags，然后在每个目录下面都新建一个index.md的文件。
tags目录下的index.md文件内容为：

title: tags
date: 2015-10-20 06:49:50
type: “tags”
comments: false

categories目录下的index.md文件的内容为：

title: categories
date: 2015-10-20 06:49:50
type: “categories”
comments: false

这样当编译的时候就会自动生成这两个总目录文件。

对于About页面也是一样，在source下面新建一个about目录，然后在目录里新建一个index.md再进行编辑或者直接用命令 

`hexo new page about `

都是OK的。

生成时遇到的问题

当执行命令 

`hexo g -d`

时出错

Unhandled rejection TypeError: path.substring is not a function
at Object.urlForHelper (C:\Users\ThinkPad\Documents\GitHub\macroblog\node_modules\hexo\lib\plugins\helper\url_for.js:9:31)
…

最后排查发现是因为在主题的设置中将rss 设置为了true，就会一直出现这个问题，当设置回false之后就又OK了。

部署

建站完成之后我们还需要将网站部署到github上面，这个时候在项目的配置文件_config.yml中我们配置一下：

deploy:
type: git
repo: https://github.com/ChengXuchao/macroblog.git
branch: master

这样之后每次使用命令
hexo g -d
就会自动部署到github上面

安装NexT主题

Hexo是支持各种主题安装的，而NexT只是其中的一个，其他你可以到Hexo的主题库里去选自己喜欢的主题，因为我目前选用的就是NexT主题所以这里就着重介绍这个主题的安装。

NexT主题官网地址：http://theme-next.iissnan.com/

NexT的安装在官网上也有非常详细的说明，官方文档地址：http://theme-next.iissnan.com/getting-started.html

将主题文件下载下来之后，拷贝到站点目录下的 themes 目录下，然后再在站点的配置文件_config.yml中修改一下配置：

theme: next

即可。

GitHub

申请注册账号

GitHub官网：https://github.com
申请注册一个账号就可以了。

建立一个代码仓库

进入到设置页面，找到 GitHub Pages 设置选项，选一个Source和Theme，这个时候代码仓库就变成了可以访问的网页了，就会为你分配一个地址，否则是不会有访问地址的，比如这个时候我的访问地址就是：
https://chengxuchao.github.io/macroblog/

此时我的个人主页就基本完成了，可以正常的访问和更新了。那么这个时候还有最后一步，就是如果你有一个域名的话就可以把域名直接映射上来。

映射域名

将域名的CNAME映射到你的github的page项目上，比如我的地址是：chengxuchao.github.io。我的域名是macrocheng.me.
在项目的根目录新建一个文件CNAME没有后缀（在本地的source目录下），里面内容就是你的域名macrocheng.me.
同时在网站的配置文件_config.yml中设置：

url: http://macrocheng.me
root: /

映射完之后，不管是通过macrocheng.me还是www.macrocheng.me都能直接访问到我的主页。

到此整个配置就结束了，现在大家都可以通过macrocheng.me 来访问我的个人博客咯！是不是很酷？
